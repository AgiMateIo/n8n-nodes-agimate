## Секция 2: Три ноды

**Заголовок секции:**
```
Что внутри
```

**Три карточки:**

### 1. ⚡ Agimate Trigger

**Подзаголовок:** Получайте события в реальном времени

**Описание:**
```
Webhook-триггер, который запускает воркфлоу при событиях
из AgiMate — сообщения, уведомления, действия на устройствах.
```

**Возможности (список):**
- Фильтрация по типам событий — получайте только то, что нужно
- Поддержка wildcard (`*`) — или конкретные типы: `mobile.trigger`, `connector.event`, `device.action`
- Авторизация по Bearer-токену — безопасное подключение
- Опционально — заголовки и query-параметры в выходных данных

**Пример use-case:**
```
Новый заказ на Ozon → Agimate Trigger → Уведомление в Telegram
```

---

### 2. 🔌 Agimate Connectors

**Подзаголовок:** Подключайтесь к 100+ сервисам

**Описание:**
```
Выполняйте методы облачных коннекторов AgiMate прямо из n8n.
Ozon, Wildberries, 1С и другие сервисы — через единый интерфейс.
```

**Возможности (список):**
- Динамический список коннекторов — выбирайте из доступных
- Каскадные настройки: коннектор → учётные данные → метод
- Описание параметров и тела запроса прямо в интерфейсе ноды
- Работает как AI-инструмент (n8n Tools) — подключайте к AI-агентам

**Пример use-case:**
```
Таймер (каждый час) → Agimate Connectors (проверить статусы заказов) → IF (новый статус) → Google Sheets
```

---

### 3. 📱 Agimate Mobile

**Подзаголовок:** Управляйте устройствами из воркфлоу

**Описание:**
```
Отправляйте команды на телефоны и компьютеры,
зарегистрированные в AgiMate. Скриншоты, TTS,
системные команды — всё из n8n.
```

**Возможности (список):**
- Список зарегистрированных устройств прямо в ноде
- Динамические действия — зависят от возможностей устройства
- JSON-тело запроса для сложных команд
- Работает как AI-инструмент (n8n Tools) — подключайте к AI-агентам

**Пример use-case:**
```
Webhook (критичный алерт) → AI-анализ → Agimate Mobile (голосовое уведомление на телефон)
```

## Секция 3: Как подключить

**Заголовок:**
```
Настройка за 3 шага
```

**Шаги (вертикально на mobile, горизонтально на desktop):**

### Шаг 1: Установите ноду

```
npm install n8n-nodes-agimate
```

Или через Community Nodes в интерфейсе n8n:
`Settings → Community Nodes → Install → n8n-nodes-agimate`

### Шаг 2: Добавьте API-ключ

Создайте credentials в n8n:
- **API URL** — адрес вашего AgiMate-сервера (по умолчанию `https://api.agimate.io`)
- **API Key** — ключ из личного кабинета AgiMate

### Шаг 3: Используйте

Перетащите любую из трёх нод в ваш воркфлоу.
Коннекторы, устройства и методы подгружаются автоматически.

## Секция 4: Примеры воркфлоу

**Заголовок:**
```
Готовые сценарии
```

**Карточки (сетка 2x3 на desktop, 1 колонка на mobile):**

### 1. 🤖 AI-секретарь с реальными действиями

```
Agimate Trigger (входящее сообщение)
  → OpenAI (анализ и генерация ответа)
  → IF (требует действия?)
    → Да: Agimate Connectors (выполнить действие)
    → Нет: Telegram (отправить ответ)
```

**Описание:** AI не просто отвечает, а действует — создаёт задачи, обновляет CRM, отвечает клиентам.

---

### 2. 📦 Мониторинг маркетплейсов

```
Schedule Trigger (каждые 15 минут)
  → Agimate Connectors (Ozon: получить заказы)
  → Agimate Connectors (WB: получить заказы)
  → Merge
  → IF (новые заказы?)
    → Google Sheets (записать)
    → Telegram (уведомление)
```

**Описание:** Автоматический сбор заказов с нескольких маркетплейсов в одну таблицу.

---

### 3. 🖥 Мониторинг рабочей станции

```
Agimate Trigger (device.action)
  → OpenAI (анализ события)
  → IF (критичное?)
    → Agimate Mobile (голосовой алерт)
    → Telegram (текстовый алерт)
```

**Описание:** Следите за событиями на компьютере. Критичные алерты — голосом на телефон.

---

### 4. 📄 Автообработка документов

```
Agimate Trigger (новый файл)
  → OpenAI (распознать тип документа, извлечь данные)
  → Switch (тип документа)
    → Счёт: Agimate Connectors (1С → создать документ)
    → Договор: Google Drive (переместить в папку)
    → Прочее: Telegram (уведомление)
```

**Описание:** Новые файлы на устройстве автоматически сортируются и обрабатываются.

---

### 5. 🔊 Умные голосовые уведомления

```
Любой триггер
  → OpenAI (приоритизация: критичное / важное / информация)
  → Switch (приоритет)
    → Критичное: Agimate Mobile (TTS на телефон + компьютер)
    → Важное: Agimate Mobile (TTS на телефон)
    → Информация: Telegram (текстом)
```

**Описание:** AI определяет важность события и выбирает канал уведомления.

---

### 6. 📊 Legacy-интеграция через устройство

```
Webhook (запрос из CRM)
  → Agimate Mobile (открыть 1С, выполнить действие)
  → Wait (5 сек)
  → Agimate Mobile (сделать скриншот)
  → OpenAI (проверить результат по скриншоту)
  → Webhook Response (результат)
```

**Описание:** Управляйте десктопными приложениями без API. AI проверяет результат по скриншоту.

## Секция 6: Типы событий

**Заголовок:**
```
Какие события можно отслеживать
```

**Таблица:**

| Тип события | Описание | Пример |
|---|---|---|
| `mobile.trigger` | События с мобильных и десктопных устройств | Новый файл, уведомление, скриншот |
| `connector.event` | События из облачных коннекторов | Новый заказ, изменение статуса, сообщение |
| `credential.updated` | Обновление учётных данных | Обновлён токен доступа к сервису |
| `device.action` | Действия на устройствах | Выполнена команда, результат действия |
| `*` | Все события | Для дебага или универсальных воркфлоу |
